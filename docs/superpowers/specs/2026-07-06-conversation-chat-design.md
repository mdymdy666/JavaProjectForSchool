# 商品会话与消息中心聊天化设计

## 1. 目标

把当前“手填接收方用户 ID、商品 ID”的留言表单升级为可直接使用的商品会话体验：商品详情页提供联系卖家弹窗，消息中心提供会话列表与聊天窗口。两处复用同一聊天面板和消息规则，不修改现有消息表结构。

## 2. 范围

本轮包含：

- 商品详情页点击“联系卖家”后打开聊天弹窗，不再直接跳转消息中心。
- 弹窗自动绑定当前商品与卖家，展示该会话历史消息并支持连续发送。
- 消息中心将交易留言改为“会话列表 + 聊天窗口”。
- 系统通知继续保留独立标签页。
- 会话未读、发送状态、错误反馈、移动端交互和轻量轮询。

本轮不包含：

- WebSocket、语音、图片消息、文件传输、撤回、删除消息。
- 新增数据库表或改变消息持久化结构。
- 跨商品的私人聊天；每条消息仍必须关联商品。

## 3. 会话模型

前端根据现有 `SiteMessage[]` 派生会话，不新增后端会话实体。

- 会话唯一键：`counterpartId:productId`。
- 当前用户是发送方时，对方为 `receiverId/receiverNickname`。
- 当前用户是接收方时，对方为 `senderId/senderNickname`。
- 同一用户针对不同商品形成不同会话。
- 会话按最后一条消息时间倒序排列。
- 未读数只统计当前用户收到且 `readStatus === 'UNREAD'` 的消息。
- 商品被删除后，沿用后端返回的“商品已删除”标题并保留历史消息。

## 4. 组件边界

### 4.1 `ChatPanel`

可复用的聊天主体组件，负责：

- 渲染左右消息气泡、昵称、时间和空状态。
- 自己发送的消息在右侧，对方消息在左侧。
- 底部输入框、500 字计数、发送按钮和发送中状态。
- `Enter` 发送、`Shift+Enter` 换行。
- 发送失败时保留输入内容并展示后端错误。
- 新消息渲染后滚动到底部。

组件通过 props 接收当前用户、会话信息和消息数组，通过事件请求发送与标记已读，不直接耦合页面路由。

### 4.2 `ProductChatDialog`

商品详情页的模态容器，负责：

- 展示卖家头像首字、昵称与关联商品名称。
- 打开时加载全部消息并筛选当前 `sellerId + productId` 会话。
- 将当前会话收到的未读消息标为已读。
- 提供关闭按钮和“前往消息中心”入口。
- 点击遮罩或按 `Escape` 关闭；发送过程中不丢失草稿。

### 4.3 消息中心

- 桌面端左侧固定宽度会话列表，右侧为 `ChatPanel`。
- 会话项展示头像首字、对方昵称、商品名、最后一条消息、时间和未读角标。
- 默认选中最近会话；支持通过 URL 查询参数 `counterpartId`、`productId` 定位会话。
- 移动端先显示会话列表，选择后进入聊天视图，并提供返回会话列表按钮。
- 系统通知仍在独立标签页中展示与批量已读。

## 5. 数据流

1. 页面调用现有 `GET /api/messages` 获取当前用户相关消息。
2. 前端按对方用户与商品分组，生成会话摘要。
3. 选择会话后筛选消息并按创建时间正序排列。
4. 发送时调用现有 `POST /api/messages`，固定传递 `receiverId`、`productId`、`content`。
5. 发送成功后把返回消息合入列表并重新获取消息，确保以后端结果为准。
6. 打开会话时，对当前用户收到的未读消息逐条调用现有已读接口。
7. 弹窗或消息中心处于可见状态时每 5 秒轻量刷新；离开页面或关闭弹窗时清理定时器。

## 6. 状态与异常

- 未登录点击联系卖家：跳转登录页，并携带当前商品页作为 `redirect`。
- 空消息：显示“还没有消息，向卖家问问商品细节吧”。
- 加载失败：聊天区域显示重试按钮，不用空白页面代替错误。
- 发送失败：优先展示后端返回的敏感词、无权限或商品不存在信息；草稿保留。
- 重复提交：发送期间禁用发送按钮。
- 对方或商品信息异常：保留历史记录，使用后端兜底名称。
- 页头未读数在读取或发送后调用通知 Store 刷新。

## 7. 视觉与响应式

- 弹窗最大宽度约 640px，高度受视口限制，消息区独立滚动。
- 气泡圆角不超过 8px；己方使用品牌蓝浅色背景，对方使用中性灰白背景。
- 商品信息位于聊天标题区，不嵌套额外卡片。
- 输入区固定在聊天底部，按钮使用明确的发送图标或“发送”文字。
- 桌面端消息中心保持工作台式双栏布局；小于 700px 时切换单栏。
- 所有按钮与文本显式设置前景色，避免再次继承全局白色按钮样式。

## 8. 测试与验收

前端自动测试：

- 会话按 `counterpartId + productId` 正确分组和排序。
- 当前用户发送/接收消息时，对方识别和气泡方向正确。
- 商品详情点击联系卖家打开弹窗，并自动传递卖家与商品 ID。
- 输入快捷键、发送中、成功清空、失败保留草稿行为正确。
- 打开会话后只标记当前用户收到的未读消息。
- 移动端会话列表与聊天视图可切换。

后端回归：

- 现有消息发送、权限限制、敏感词过滤和已读测试继续通过。
- 不改变消息接口响应字段和数据库结构。

人工验收：

1. 买家从商品详情打开聊天弹窗，发送咨询。
2. 切换卖家账号，在消息中心看到未读会话并回复。
3. 买家重新打开商品弹窗看到完整对话。
4. 页头未读角标随读取状态变化。
5. 在窄屏下完成选择会话、发送和返回列表操作。
