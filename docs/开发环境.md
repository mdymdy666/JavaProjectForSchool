## 自动检测脚本

当前仓库提供依赖检测脚本：

```text
scripts/check-dependencies.bat
```

运行后生成：

```text
reports/dependency-check-report.md
```

该脚本用于检测不同电脑上的工具版本、命令来源、安装路径、MySQL/Redis 服务状态，以及后续正式工程中的 Maven / Node 配置型依赖。脚本只做检测和报告生成，不修改系统 PATH，不安装软件，不修改 MySQL、Redis 或 Java 配置。

## 约束

1. Java、MySQL、Redis 作为关键环境，不在未确认情况下由 Agent 擅自安装。
2. 新安装的非关键依赖如确需安装，应放在 `D:\soft_wares5\依赖名称\依赖文件`。
3. 不把第三方运行时、数据库数据目录、Redis 数据文件、安装包提交到 Git。
4. 修改系统 PATH 或用户 PATH 前必须说明修改内容，并优先考虑局部配置。

## 验收标准

- 本机工具版本有实际检测来源。
- 项目依赖没有凭空填写版本。
- MySQL 的 PATH 不可用与服务存在状态被分别记录。
