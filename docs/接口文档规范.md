# 接口文档规范

## 背景

本项目后续采用前后端分离架构，前端 Vue3 与后端 Spring Boot 需要通过 RESTful API 进行数据交互。接口文档是前后端并行开发、联调和测试的基础。

## 目标

统一接口描述格式，减少接口字段、权限、错误码和响应结构的沟通成本。

## 范围

适用于后续所有用户管理、商品管理、订单交易、消息沟通、后台管理、数据统计、公告管理等接口。

## 基本要求

每个接口应至少包含：

| 字段 | 说明 |
|---|---|
| 接口名称 | 简明描述接口用途 |
| 请求方法 | GET、POST、PUT、PATCH、DELETE 等 |
| 请求路径 | 例如 `/api/products` |
| 权限要求 | 游客、登录用户、卖家、管理员等 |
| 请求参数 | Path、Query、Header、Body 参数 |
| 响应结构 | 成功响应字段与数据结构 |
| 错误码 | 业务错误码、HTTP 状态码、错误说明 |
| 示例 | 请求示例和响应示例 |

## 接口设计约束

1. API 路径建议统一以 `/api` 开头。
2. 认证接口后续应明确 JWT 的请求头格式。
3. 后台管理接口应与普通用户接口区分权限。
4. 文件上传接口应说明文件大小、类型、数量、存储方式和返回 URL 规则。
5. 分页接口应统一分页参数和返回结构。
6. 错误响应结构应保持一致。

## 验收标准

- 前端可根据接口文档完成调用代码。
- 后端可根据接口文档实现 Controller 和 DTO。
- 测试人员可根据接口文档编写接口测试用例。
- 接口权限、参数、响应、错误码清晰完整。
