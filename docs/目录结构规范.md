# 目录结构规范

## 背景

项目后续将采用前后端分离架构。为避免业务代码、文档、配置、脚本和运行时文件混杂，需要在开发开始前约定目录结构。

## 目标

保证项目结构清晰，便于多人协作、文档维护、前后端联调和 Git 管理。

## 当前目录结构

```text
JavaSpringProject/
  README.md
  codex.md
  docs/
  徐士伟-软件与理论综合设计与实践项目指导书【徐士伟-题目2-2026】.docx
```

## 后续推荐结构

```text
JavaSpringProject/
  backend/                  # Spring Boot 后端项目，后续创建
  frontend/                 # Vue3 前端项目，后续创建
  docs/                     # 项目文档
  scripts/                  # 辅助脚本，例如本地启动、数据库初始化
  sql/                      # 数据库设计和初始化脚本
  README.md
  codex.md
  .gitignore
  .gitattributes
```

## 目录职责

| 目录 | 职责 | 当前状态 |
|---|---|---|
| `backend/` | Spring Boot 后端代码、配置、测试 | 未创建 |
| `frontend/` | Vue3 前端代码、组件、页面、测试 | 未创建 |
| `docs/` | 项目说明、规范、设计、测试、环境文档 | 已创建 |
| `scripts/` | 可复用辅助脚本 | 未创建 |
| `sql/` | 数据库 DDL、初始化数据、变更脚本 | 未创建 |

## 约束

1. 不把 Node 依赖目录 `node_modules/` 提交到 Git。
2. 不把 Java 构建产物 `target/`、`build/`、`out/` 提交到 Git。
3. 不把上传文件、日志、缓存、数据库数据、Redis 数据文件提交到 Git。
4. 不把第三方运行时或大型安装包放入项目目录。
5. 后续如需本地配置，应提供样例文件，不提交真实敏感配置。

## 验收标准

- 每类文件有清晰归属。
- 新增目录与本规范不冲突。
- 运行时文件和敏感信息不会进入 Git。
