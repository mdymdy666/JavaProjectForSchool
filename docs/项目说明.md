# 项目说明

## 背景

本项目名称为“校园二手物品交易平台的设计与实现”，项目类型为应用系统，来自《徐士伟-软件与理论综合设计与实践项目指导书【徐士伟-题目2-2026】》。

指导书要求项目基于 Vue3 + Spring Boot + MySQL + Redis 技术栈，采用前后端分离架构，实现校园 C2C 二手物品交易 Web 应用。

## 目标

后续开发计划实现一个功能完善、界面友好、安全可靠的校园二手交易平台，支持用户注册登录、商品发布浏览、订单交易、买卖沟通、后台审核、数据统计和公告管理等能力。

## 范围

计划包含以下模块：

| 模块 | 项目要求 |
|---|---|
| 用户管理 | 注册、登录、密码找回、个人中心、头像上传、个人信息编辑、收货地址管理，信誉评分与实名认证为可选项 |
| 商品管理 | 商品发布、多图上传、分类选择、价格设定、富文本描述、浏览、搜索、排序、详情、收藏、浏览量统计、状态管理 |
| 订单交易 | 直接购买或购物车、订单生成、订单状态流转、支付接口模拟或沙箱、物流信息模拟 |
| 消息沟通 | 站内信或留言板、系统通知、已读未读标记 |
| 后台管理 | 用户管理、商品审核、订单管理、数据统计、公告管理 |
| 扩展功能 | 商品推荐、敏感词过滤、内容审核、交易纠纷举报与处理等可选功能 |

## 技术路线

| 层级 | 项目要求或计划 |
|---|---|
| 前端 | Vue3，后续可结合 Vite、Axios、状态管理、UI 组件库、ECharts |
| 后端 | Spring Boot，后续配置 Spring Security、JWT、统一响应、统一异常、文件上传 |
| 数据库 | MySQL，后续设计用户、商品、订单、消息、分类、公告等表 |
| 缓存 | Redis，后续用于验证码、热点商品、用户会话或消息通知辅助能力 |
| 架构 | 前后端分离，RESTful API，响应式界面，PC 端主流浏览器兼容 |
